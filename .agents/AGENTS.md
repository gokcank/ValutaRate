## Git Rules
- Always provide clear, short, and descriptive commit messages for every commit.

## Task Tracking
- ALWAYS check the scratch_notes.md file in the root directory for new tasks, notes, or ideas from the user. It is the primary source of truth for upcoming work.

## Strict Operational Boundaries
- **No File Copying outside Workspace:** DO NOT copy APKs, assets, or any files to the user's Desktop or anywhere outside the project scope unless explicitly ordered. The user will handle testing via the 'build/outputs/' directories.
- **No Presumptuous Commands:** DO NOT bundle commands based on what you *assume* the user wants to do next. Wait for the user's instructions. You are strictly forbidden from taking the initiative on workflow decisions.
- **No Automatic Scope Inflation:** DO NOT add backlog tasks or scratchpad ideas to an active implementation plan unless the user explicitly tells you to include them in that specific plan.
